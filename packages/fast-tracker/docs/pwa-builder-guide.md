# PWA Builder Guide for Time Tracker

This guide provides step-by-step instructions for using PWA Builder to package the Time Tracker PWA as native apps for Android and iOS.

## Preparing Your PWA

Before using PWA Builder, ensure your PWA meets these requirements:

1. **Valid Web Manifest**: The `manifest.json` file should include:
   - `name` and `short_name`
   - `icons` in various sizes (at least 192x192 and 512x512)
   - `start_url`
   - `display` set to "standalone"
   - `theme_color` and `background_color`

2. **Service Worker**: A properly registered service worker for offline functionality

3. **HTTPS**: The PWA must be hosted on a secure (HTTPS) domain

## Using PWA Builder

### Step 1: Host Your PWA

Deploy your Time Tracker PWA to a web hosting service that supports HTTPS. Options include:
- GitHub Pages
- Netlify
- Vercel
- Firebase Hosting

### Step 2: Generate App Packages

1. Visit [PWA Builder](https://www.pwabuilder.com/)
2. Enter the URL of your hosted PWA
3. Wait for the site analysis to complete
4. Review any suggestions to improve your PWA
5. Click "Next" to proceed to the packaging options

### Step 3: Android Package

1. Select the "Android" option
2. Configure package options:
   - Package ID: `com.conciseways.timetracker`
   - App name: "Time Tracker"
   - Launcher name: "Time Tracker"
   - Version code: 1
   - Version name: "1.0.0"
   - Host location: Your PWA URL
   - Status bar color: "#3498db"
   - Splash screen color: "#3498db"
   - Signing key: Generate a new key or use an existing one

3. Click "Generate" to create the Android package
4. Download the generated .apk or .aab file

### Step 4: iOS Package

1. Select the "iOS" option
2. Configure package options:
   - Bundle ID: `com.conciseways.timetracker`
   - App name: "Time Tracker"
   - Version: "1.0.0"
   - Team ID: Your Apple Developer Team ID
   - Host location: Your PWA URL

3. Click "Generate" to create the iOS package
4. Download the generated package

## Publishing to App Stores

### Google Play Store

1. Create a [Google Play Developer account](https://play.google.com/console/signup) ($25 one-time fee)
2. Create a new app in the Google Play Console
3. Fill in the app details, store listing, and content rating
4. Upload the .aab file generated by PWA Builder
5. Set up pricing and distribution
6. Submit for review

### Apple App Store

1. Enroll in the [Apple Developer Program](https://developer.apple.com/programs/) ($99/year)
2. Create a new app in App Store Connect
3. Configure app information, pricing, and availability
4. Upload the build using Xcode or Application Loader
5. Submit for review

## Testing Before Submission

Before submitting to app stores:

1. Install the generated packages on test devices
2. Verify all functionality works as expected
3. Test offline capabilities
4. Check that app icons and splash screens display correctly
5. Test on different device sizes and orientations

## Troubleshooting

If you encounter issues with PWA Builder:

1. **Manifest Issues**: Validate your manifest using [Manifest Validator](https://manifest-validator.appspot.com/)
2. **Service Worker**: Check browser developer tools for service worker registration errors
3. **HTTPS**: Ensure your site is properly served over HTTPS
4. **Icons**: Verify all required icon sizes are available and properly referenced in the manifest
